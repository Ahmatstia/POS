04 — ANTI-GENERIC RULES (CRITICAL)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
NEVER use any of the following — they signal generic AI output:
  ✗ Top app bar / toolbar with hamburger menu icon
  ✗ Bottom navigation bar (this is a tablet app, use side nav)
  ✗ Material blue (#2196F3) or teal anywhere
  ✗ Generic card with just text + a chevron arrow (ListTile pattern)
  ✗ Pie chart or donut chart as primary data visualization
  ✗ Full-page modal dialogs for simple confirmations
  ✗ Lorem ipsum text — use real realistic dummy data in Indonesian
  ✗ Uniform white product cards with no visual hierarchy
  ✗ Generic "icon in a colored circle" card layout (the DashCard pattern)
  ✗ Tables with no alternating rows and no column sorting affordance
  ✗ Form fields stacked uniformly with no section grouping

ALWAYS do the following:
  ✓ Left sidebar navigation rail with active state indicator pill
  ✓ Product images with realistic placeholder (gray gradient aspect-ratio box)
  ✓ Stock badges: colored pill positioned absolute top-right of product card
  ✓ Data tables: alternating #F8FAFC / white rows, sort icon on header
  ✓ Empty states: illustration area + clear headline + CTA button
  ✓ Status badges: pill shape with colored bg + matching text (not just colored text)
  ✓ Numbers/amounts: always right-aligned in tables
  ✓ Use realistic dummy data: Indonesian product names, store name, cashier name
